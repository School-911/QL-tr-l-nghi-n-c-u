# 🚀 Hướng Dẫn AI Core - Phân Tích & Lập Báo Cáo

## 📋 Yêu Cầu Quy Tắc Tuân Thủ

Đọc file `.agent.md` để hiểu đầy đủ các quy tắc tuyệt đối phải tuân thủ.

---

## 🔄 Quy Trình Xử Lý Yêu Cầu

### Phase 1: Tiếp Nhận & Phân Tích
```
Input:
- user_query: Câu hỏi/yêu cầu từ người dùng
- target_url: URL hoặc đường dẫn file

Output:
- research_objective: Mục tiêu phân tích được xác định
- session_id: ID phiên làm việc
```

### Phase 2: Trích Xuất Dữ Liệu
```
Hành động:
1. Kết nối tới URL hoặc mở file
2. Trích xuất toàn bộ nội dung (KHÔNG lấy mẫu)
3. Kiểm tra dữ liệu trả về

Xử lý Lỗi:
- Nếu TRỐNG/LỖI: DỪNG → Ghi lỗi vào MongoDB
- Nếu THÀNH CÔNG: Tiếp tục Phase 3
```

### Phase 3: Phân Tích & Tổng Hợp
```
Hành động:
1. Phân tích nội dung THỰC TẾ từ dữ liệu trích xuất
2. Tìm kiếm ý chính, kết quả
3. Lập bảng phân tích dựa trên nội dung thực

QUY TẮC:
- KHÔNG được dùng dữ liệu cứng
- Tất cả điểm phải từ nguồn dữ liệu
- Báo cáo viết 100% Tiếng Việt
```

### Phase 4: Xuất Báo Cáo
```
Format bắt buộc:
BÁO CÁO NGHIÊN CỨU & TỔNG HỢP KIẾN THỨC
Mục tiêu nghiên cứu: [mục tiêu]
Nguồn dữ liệu: [URL/tệp]

📌 1. Tổng quan & Tóm tắt...
💡 Các phát hiện cốt lõi:...
📌 2. Chi tiết Cấu trúc...
📌 3. Khuyến nghị...

[Hệ thống] Trạng thái: Đã đồng bộ và lưu vào MongoDB thành công.
```

### Phase 5: Lưu Lịch Sử
```
Gọi API Backend:
POST /api/research/history
{
  "user_query": "...",
  "target_url": "...",
  "research_objective": "...",
  "status": "success",
  "response_content": "[toàn bộ báo cáo]",
  "session_id": "...",
  "processing_time_ms": ...
}
```

## 🚨 Xử Lý Lỗi

### Tình Huống 1: Không Kết Nối Được URL
```
In ra:
[Lỗi] Không thể kết nối tới URL. Vui lòng kiểm tra lại đường dẫn.

Lưu vào MongoDB:
{
  "status": "failed",
  "error_code": "CONNECTION_ERROR",
  "response_content": "[Lỗi] Không thể kết nối tới URL..."
}
```

### Tình Huống 2: Model LLM Hết Token
```
In ra:
[Lỗi] Model LLM hết token. Vui lòng thử lại sau.

Lưu vào MongoDB:
{
  "status": "failed",
  "error_code": "TOKEN_LIMIT_EXCEEDED",
  "response_content": "[Lỗi] Model LLM hết token..."
}
```

### Tình Huống 3: File Trống/Dữ Liệu Không Hợp Lệ
```
In ra:
[Lỗi] Không thể lấy nội dung từ nguồn dữ liệu.

Lưu vào MongoDB:
{
  "status": "failed",
  "error_code": "EMPTY_CONTENT",
  "response_content": "[Lỗi] Không thể lấy nội dung..."
}
```

## 🔗 Tích Hợp Backend

### Cấu Hình
```python
# Trong app.py
BACKEND_API_URL = os.getenv("BACKEND_API_URL", "http://localhost:5000")
RESEARCH_HISTORY_ENDPOINT = f"{BACKEND_API_URL}/api/research/history"
```

### Gọi API Lưu Lịch Sử
```python
import requests
import time

def save_to_mongodb(user_query, target_url, research_objective, status, response_content, session_id, processing_time):
    payload = {
        "user_query": user_query,
        "target_url": target_url,
        "research_objective": research_objective,
        "status": status,
        "response_content": response_content,
        "session_id": session_id,
        "processing_time_ms": processing_time
    }
    
    try:
        response = requests.post(RESEARCH_HISTORY_ENDPOINT, json=payload)
        if response.status_code == 201:
            print("[✓] Lịch sử đã được lưu vào MongoDB thành công")
            return True
        else:
            print(f"[!] Cảnh báo: Không thể lưu vào MongoDB (Status: {response.status_code})")
            return False
    except Exception as e:
        print(f"[!] Cảnh báo: Lỗi khi gọi API Backend: {str(e)}")
        return False
```
