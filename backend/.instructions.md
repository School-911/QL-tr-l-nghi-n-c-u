# 📌 Hướng Dẫn Backend - API Research Assistant

## Tích Hợp MongoDB Research History

### 1. Model Schema
File: `src/models/researchHistory.model.js`

Lưu trữ lịch sử tất cả các phiên phân tích/nghiên cứu theo cấu trúc:
```javascript
{
  user_query: String,        // Câu hỏi gốc
  target_url: String,        // URL/Tệp nguồn
  research_objective: String, // Mục tiêu phân tích
  status: 'success'|'failed', // Trạng thái
  response_content: String,   // Nội dung báo cáo/lỗi
  error_code: String,         // Mã lỗi (nếu có)
  error_details: String,      // Chi tiết lỗi (nếu có)
  user_id: ObjectId,          // User ID
  session_id: String,         // Session ID từ AI Core
  processing_time_ms: Number, // Thời gian xử lý
  metadata: Object,           // Metadata bổ sung
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 2. API Endpoint Ghi Nhận Lịch Sử

**Endpoint**: `POST /api/research/history`

**Request Body**:
```json
{
  "user_query": "Phân tích về AI trong y tế",
  "target_url": "https://example.com/article.pdf",
  "research_objective": "Tìm hiểu các ứng dụng AI trong lĩnh vực y tế",
  "status": "success",
  "response_content": "BÁO CÁO NGHIÊN CỨU...",
  "session_id": "uuid-session-id",
  "processing_time_ms": 1250,
  "metadata": {
    "pdf_pages": 45,
    "keyword_count": 12,
    "model_used": "Gemini 1.5 Pro"
  }
}
```

**Response**:
```json
{
  "status": "success",
  "message": "Lịch sử phiên làm việc đã được lưu thành công",
  "data": {
    "_id": "...",
    "user_query": "...",
    "status": "success",
    "createdAt": "2026-05-22T00:21:44Z"
  }
}
```

### 3. Query Lịch Sử

**Endpoint**: `GET /api/research/history`

**Query Parameters**:
- `status`: 'success' | 'failed'
- `user_id`: ID người dùng
- `page`: Trang (default: 1)
- `limit`: Số lượng/trang (default: 20)

**Response**:
```json
{
  "status": "success",
  "data": [
    {
      "_id": "...",
      "user_query": "...",
      "research_objective": "...",
      "status": "success",
      "created_at": "..."
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150
  }
}
```

### 4. Xóa Lịch Sử Cũ

**Endpoint**: `DELETE /api/research/history/:id`

---

## Quy Tắc Tích Hợp

1. **Lưu luôn**: Mọi phiên phân tích (thành công/lỗi) phải được lưu vào MongoDB
2. **Không sửa dữ liệu gốc**: `response_content` phải lưu nguyên như được trả về từ AI Core
3. **Thêm timestamp**: Tự động ghi `createdAt` khi insert
4. **Xử lý lỗi**: Nếu ghi MongoDB thất bại, log error nhưng vẫn trả response về user
5. **Bảo mật**: Chỉ user sở hữu mới có thể query lịch sử của họ (nếu có auth)
